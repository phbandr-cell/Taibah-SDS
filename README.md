# Taibah SDS Management

بوابة خفيفة للبحث في ملفات Safety Data Sheets الخاصة بكلية الصيدلة في جامعة طيبة.

## التشغيل المحلي

```powershell
python -m pip install -r requirements.txt
python scripts/build_sds_hazards.py
python -m uvicorn main:app --reload --host 127.0.0.1 --port 8000
```

بعد التشغيل افتح:

```text
http://127.0.0.1:8000
```

## المعمارية الحالية

- `main.py`: تطبيق FastAPI، يقدم الصفحة الرئيسية، الملفات الثابتة، البحث، الإحصاءات، وفحص الصحة.
- `static/index.html`: واجهة عربية/إنجليزية لعرض البحث، التصنيفات، الإحصاءات، وروابط ملفات PDF.
- `static/pdfs/Safety_Data_Sheets`: مصدر بيانات النظام الحالي، ويعتمد البحث على أسماء ملفات PDF.
- `vercel.json`: إعداد نشر Vercel باستخدام Python runtime.

## نقاط النهاية

- `GET /`: الواجهة الرئيسية.
- `GET /health`: حالة التطبيق وعدد ملفات SDS.
- `GET /api/stats`: عدد الملفات، الحجم الإجمالي، والتصنيفات.
- `GET /api/sds?limit=60&offset=0`: قائمة ملفات SDS.
- `GET /api/search-sds?q=ethanol&limit=40`: البحث الحديث.
- `GET /search-sds/{query}`: مسار توافق قديم.

## رموز السلامة

يعرض النظام مؤشرات GHS تحذيرية مع كل نتيجة بحث، مثل قابل للاشتعال، آكل، سام، مهيج، مؤكسد، خطر صحي، وخطر بيئي.

مصدر الرموز بالترتيب:

- أكواد `GHS01` إلى `GHS09` الموجودة نصيًا داخل ملف PDF.
- عبارات المخاطر `H` داخل ملف PDF، مثل `H225` أو `H314`، ثم تحويلها إلى رمز GHS مناسب.
- قواعد تقديرية من اسم الملف عند عدم استخراج أي رمز أو عبارة من PDF.

يولّد السكربت `scripts/build_sds_hazards.py` ملف `static/data/sds_hazards.json` من ملفات PDF. المرجع النهائي لاتخاذ قرار السلامة يجب أن يكون وثيقة SDS المعتمدة نفسها.

## عصف ذهني للتطوير

### بيانات SDS

- إنشاء ملف `sds_manifest.csv` أو قاعدة بيانات تحتوي على الاسم الرسمي، المرادفات، CAS Number، المورد، تاريخ الإصدار، تاريخ آخر مراجعة، ورابط الملف.
- إضافة حالة اعتماد لكل وثيقة: مسودة، معتمدة، تحتاج مراجعة، منتهية.
- تسجيل تاريخ رفع الملف ومن قام بمراجعته.
- ربط أكثر من اسم تجاري أو مرادف بنفس المادة.
- استخراج نصوص PDF وفهرستها للبحث داخل محتوى الوثيقة، وليس اسم الملف فقط.
- إدخال رموز GHS الرسمية لكل مادة في ملف manifest بدل الاعتماد على الاستنتاج من الاسم.

### البحث والتجربة

- دعم البحث بالعربية والإنجليزية والمرادفات.
- دعم CAS Number.
- ترتيب النتائج حسب قوة المطابقة والأولوية.
- إضافة فلاتر: التصنيف، المورد، تاريخ المراجعة، حالة الاعتماد.
- إضافة صفحة تفاصيل لكل مادة تعرض ملخصًا منظمًا قبل فتح PDF.
- إضافة QR code لكل ملف لاستخدامه على ملصقات المختبر.

### السلامة والامتثال

- عدم اعتبار التصنيف المستنتج من الاسم بديلًا عن SDS.
- إضافة تنبيه عند الوثائق القديمة أو غير المعتمدة.
- إضافة سجل مراجعة audit log.
- إضافة سياسة صلاحيات: قارئ، مشرف مختبر، مسؤول SDS.
- توثيق إجراءات التحديث والمراجعة الدورية.

### التشغيل والنشر

- نقل ملفات PDF الكبيرة إلى CDN أو Object Storage.
- إضافة CI يفحص الروابط، صيغة manifest، وتوفر كل ملف PDF.
- تثبيت إصدارات الحزم، وهذا مفعّل حاليًا في `requirements.txt`.
- إضافة مراقبة صحة `/health`.
- إضافة نسخ احتياطي دوري للملفات والبيانات الوصفية.

### الأمن

- إضافة Security Headers.
- منع رفع أو تعديل الملفات من الواجهة العامة دون مصادقة.
- فحص أسماء الملفات والروابط عند أي عملية رفع مستقبلية.
- تحديد ما إذا كانت قائمة المواد يجب أن تكون عامة أم داخل شبكة الجامعة فقط.

### الاختبارات

- اختبار الصفحة الرئيسية.
- اختبار `/api/stats`.
- اختبار البحث بحالات مثل `ethanol`, `sodium`, وطلب بلا نتائج.
- اختبار أن كل نتيجة ترجع رابط PDF قابلًا للفتح.
- اختبار أن كل ملف في manifest موجود فعليًا في مجلد PDF.
